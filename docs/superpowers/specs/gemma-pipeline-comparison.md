## 关键代码差异

### routes_demo.py 当前问题

```python
# 当前: vision 和 decision 分开，且候选模型列表缺 Gemma
vision_candidates = ["llava:7b", "llava:13b", "moondream2", "llava"]   # ❌ 缺 gemma4-e2b
decision_candidates = ["deepseek-r1:1.5b", "deepseek-r1:8b"]           # ❌ 缺 gemma4-e2b

# 期望: 统一使用 gemma4-e2b
model = "gemma4-e2b"  # ✅ 一个模型处理 vision + decision
```

### analyze_pipeline.py 当前问题

```python
# 当前: 两个独立的服务
self.vision = VisionService()      # 使用 llava:7b
self.decision = DecisionService()    # 使用 gemma4-e2b

# 期望: 统一使用 Gemma4-e2b
self.model = "gemma4-e2b"  # 一个模型处理所有
```

## 优化方向

| 方向 | 当前 | 期望 | 优先级 |
|------|------|------|--------|
| 统一模型 | Vision用LLaVA, Decision用Gemma | 全部用 Gemma4-e2b | P0 |
| 简化架构 | 3个独立服务 | 1个 Gemma 服务 | P1 |
| 优化 Prompt | 两套不同 prompt | 统一端到端 prompt | P1 |
| 性能优化 | 多次 API 调用 | 单次调用 + 结构化输出 | P2 |
