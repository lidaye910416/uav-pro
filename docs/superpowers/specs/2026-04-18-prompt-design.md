# 异常检测 Prompt 设计方案

> 日期: 2026-04-18  
> 目标: 设计异常识别 Prompt 和处置规范库的无缝对接

## 一、异常识别 Prompt (阶段1)

```
┌────────────────────────────────────────────────────────────────────────────────────┐
│                         异常识别 Prompt (阶段1)                                         │
├────────────────────────────────────────────────────────────────────────────────────┤
│                                                                                     │
│  System:                                                                            │
│  ┌─────────────────────────────────────────────────────────────────────────────┐  │
│  │                                                                             │  │
│  │  你是一个专业的高速公路航拍图像分析专家。                                    │  │
│  │  你的任务是根据图像区域中的视觉特征，                                       │  │
│  │  判断是否存在以下6类道路异常事件。                                         │  │
│  │                                                                             │  │
│  │  【6类异常定义】                                                           │  │
│  │  1. collision - 交通事故/碰撞: 车辆相撞、追尾、侧翻，多车聚集            │  │
│  │  2. pothole - 路面塌陷/坑洞: 路面出现凹陷、破损、坑洞、积水区域          │  │
│  │  3. obstacle - 道路障碍物: 落石、遗撒物、施工围挡、散落货物                │  │
│  │  4. parking - 异常停车: 应急车道停车、紧急停车带停车、故障车辆           │  │
│  │  5. pedestrian - 行人闯入: 行人、动物出现在行车道或应急车道               │  │
│  │  6. congestion - 交通拥堵: 车辆密度异常高、车辆排队、行驶缓慢             │  │
│  │                                                                             │  │
│  │  【视觉判断要点】                                                         │  │
│  │  - collision: 多辆车辆聚集、车辆变形、车头相撞、散落物、刹车痕迹          │  │
│  │  - pothole: 路面颜色变暗、边缘不规则、形状不规则、可能有积水              │  │
│  │  - obstacle: 非车辆物体出现在路面、形状不规则、非道路结构物                │  │
│  │  - parking: 车辆静止停在非停车区域、车门可能打开、人员下车                 │  │
│  │  - pedestrian: 人形/动物形状、移动模式与车辆不同、出现在非正常区域       │  │
│  │  - congestion: 车辆密集排列、移动缓慢或停止、车流呈队列状                  │  │
│  │                                                                             │  │
│  │  请仔细分析图像区域，只输出你确认存在的异常类型。                          │  │
│  │                                                                             │  │
│  └─────────────────────────────────────────────────────────────────────────────┘  │
│                                                                                     │
│  User:                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────────────┐  │
│  │                                                                             │  │
│  │  【待分析图像区域】                                                        │  │
│  │  {roi_image} 或 {roi_description}                                         │  │
│  │                                                                             │  │
│  │  【运动检测信息(如有)】                                                    │  │
│  │  {motion_info}                                                            │  │
│  │                                                                             │  │
│  │  请分析以上信息，判断是否存在异常事件。                                     │  │
│  │  只输出确认存在的异常，不要臆测。                                           │  │
│  │                                                                             │  │
│  └─────────────────────────────────────────────────────────────────────────────┘  │
│                                                                                     │
│  Output Format:                                                                     │
│  ┌─────────────────────────────────────────────────────────────────────────────┐  │
│  │                                                                             │  │
│  │  {                                                                          │  │
│  │    "has_incident": true/false,                                              │  │
│  │    "incident_type": "collision/pothole/obstacle/parking/pedestrian/        │  │
│  │                       congestion/none",                                      │  │
│  │    "confidence": 0.0-1.0,                                                  │  │
│  │    "description": "具体描述观察到的异常情况(50字内)",                       │  │
│  │    "visual_evidence": ["观察到的视觉证据1", "观察到的视觉证据2"],           │  │
│  │    "reason": "判断依据(30字内)"                                            │  │
│  │  }                                                                          │  │
│  │                                                                             │  │
│  └─────────────────────────────────────────────────────────────────────────────┘  │
│                                                                                     │
└────────────────────────────────────────────────────────────────────────────────────┘
```

## 三、决策输出 Prompt (阶段2)

```
┌────────────────────────────────────────────────────────────────────────────────────┐
│                         决策输出 Prompt (阶段2) - 含RAG上下文                             │
├────────────────────────────────────────────────────────────────────────────────────┤
│                                                                                     │
│  System:                                                                            │
│  ┌─────────────────────────────────────────────────────────────────────────────┐  │
│  │                                                                             │  │
│  │  你是一个道路安全应急指挥专家。                                              │  │
│  │  根据异常检测结果和处置规范，                                               │  │
│  │  输出标准化的预警信息和处置建议。                                            │  │
│  │                                                                             │  │
│  │  【风险等级定义】                                                         │  │
│  │  - critical: 严重风险，需立即处置，可能造成重大事故                         │  │
│  │  - high: 高风险，需尽快处置，可能影响交通安全                               │  │
│  │  - medium: 中风险，需跟踪关注，可能逐步恶化                                 │  │

│  │                                                                             │  │
│  │  【输出要求】                                                             │  │
│  │  - title: 10字内，简明扼要                                                 │  │
│  │  - description: 30字内，描述现场情况                                       │  │
│  │  - recommendation: 40字内，具体可执行的处置建议                           │  │
│  │  - 所有建议必须来自RAG检索到的处置规范，不可臆造                           │  │
│  │                                                                             │  │
│  └─────────────────────────────────────────────────────────────────────────────┘  │
│                                                                                     │
│  User:                                                                             │

│  │                                                                             │  │
│  │  请基于以上信息，按照规范要求输出预警信息。                                 │  │
│  │  处置建议必须从RAG规范中提取，不要添加规范以外的内容。                     │  │
│  │                                                                             │  │
│  └─────────────────────────────────────────────────────────────────────────────┘  │
│                                                                                     │
│  Output Format:                                                                     │
│  ┌─────────────────────────────────────────────────────────────────────────────┐  │
│  │                                                                             │  │
│  │  {                                                                          │  │

## 四、Prompt 与规范库对接关系

```
┌────────────────────────────────────────────────────────────────────────────────────┐
│                         Prompt 与 规范库 对接关系                                         │
├────────────────────────────────────────────────────────────────────────────────────┤
│                                                                                     │
│  ┌─────────────────────────────────────────────────────────────────────────────┐  │
│  │                                                                             │  │
│  │   阶段1: 异常识别                                                           │  │
│  │   ┌─────────────────────────────────────────────────────────────────────┐ │  │
│  │   │ Prompt: "判断是否存在6类异常"                                        │ │  │
│  │   │ Prompt: "collision/pothole/obstacle/parking/pedestrian/congestion"   │ │  │
│  │   │ Prompt: "visual_evidence: [观察到的视觉证据]"                          │ │  │
│  │   └─────────────────────────────────────────────────────────────────────┘ │  │
│  │                                    │                                        │  │
│  │                                    ▼                                        │  │
│  │   异常类型 + 视觉证据 ───────────────────────▶ RAG检索                      │  │
│  │                                              │                              │  │
│  │                                              ▼                              │  │
│  │   ┌─────────────────────────────────────────────────────────────────────┐ │  │
│  │   │ RAG: 根据 incident_type 检索对应规范                                  │ │  │
│  │   │ RAG: 根据 visual_evidence 匹配 risk_indicators                       │ │  │
│  │   │ RAG: 检索对应处置流程 content                                        │ │  │
│  │   └─────────────────────────────────────────────────────────────────────┘ │  │
│  │                                    │                                        │  │
│  │                                    ▼                                        │  │
│  │   阶段2: 决策输出                                                           │  │
│  │   ┌─────────────────────────────────────────────────────────────────────┐ │  │
│  │   │ Prompt: "基于RAG规范输出预警"                                         │ │  │
│  │   │ Prompt: "recommendation必须来自规范"                                  │ │  │
│  │   │ Output: risk_level, title, description, recommendation              │ │  │
│  │   └─────────────────────────────────────────────────────────────────────┘ │  │
│  │                                                                             │  │
│  └─────────────────────────────────────────────────────────────────────────────┘  │
│                                                                                     │
│  【对接关键点】                                                                   │
│  ┌─────────────────────────────────────────────────────────────────────────────┐  │
│  │                                                                             │  │
│  │  1. incident_type 一一对应规范库的 type 字段                               │  │
│  │                                                                             │  │
│  │  2. visual_evidence 匹配规范库的 risk_indicators                            │  │
│  │     - 识别时收集证据                                                        │  │
│  │     - 检索时匹配指标                                                        │  │
│  │                                                                             │  │
│  │  3. recommendation 直接取自规范库的 content                               │  │
│  │     - 不臆造处置建议                                                        │  │
│  │     - 从规范中提取关键步骤                                                  │  │
│  │                                                                             │  │
│  │  4. risk_level 结合规范库的 priority 和异常实际情况                         │  │
