# 演示 Pipeline 可视化设计

## 1. 背景

当前首页 DemoSection 只有 4 个顺序亮起的小圆点，演示过程不直观，无法看到每个阶段的具体输出。用户无法感知"感知→识别→RAG→决策"每一阶段在做什么。

## 2. 目标

- 加载演示视频缩略图，直观展示当前使用的视频来源
- Pipeline 各阶段展开显示该阶段的实际输出内容
- 全程透明化：用户清楚知道每一阶段在做什么、输出了什么
- 保持与现有暗色设计语言一致

## 3. 整体布局

```
┌─────────────────────────────────────────────────────┐
│  控制栏（固定）                                       │
│  [视频缩略图]  视频名称标签     [▶ 启动演示]         │
├─────────────────────────────────────────────────────┤
│  Pipeline 阶段卡片区（运行后展开，4张横向排列）       │
│  [感知层] [视觉识别] [RAG检索] [决策生成]           │
├─────────────────────────────────────────────────────┤
│  Alert 结果卡片（完成后滑入）                        │
└─────────────────────────────────────────────────────┘
```

## 4. 控制栏

| 元素 | 内容 |
|------|------|
| 视频缩略图 | 来自后端 `/api/v1/demo/thumbnail`，从演示视频第10%帧提取，JPEG格式 |
| 视频名称 | 当前演示视频文件名（如 `DJI_0025_cut1.Mp4`） |
| 标签 | `4K航拍 · DJI无人机 · 高速公路`（硬编码，描述视频类型） |
| 按钮 | `▶ 启动演示`（运行时变为 `◈ 运行中...`，禁用） |

## 5. Pipeline 阶段卡片

### 5.1 4 张卡片布局

横向等宽排列，每张卡片间距 12px。

### 5.2 卡片状态

| 状态 | 表现 |
|------|------|
| **待机** | 序号圆圈灰色（#3D3D3D），内容区折叠 |
| **运行中** | 序号圆圈带脉冲动画（颜色按阶段），内容区展开显示当前输出 |
| **完成** | 序号圆圈绿色（#00E5A0）+ ✓ 标记，内容区收起，显示完成摘要 |
| **错误** | 序号圆圈红色（#FF3B3B），内容区显示错误信息 |

### 5.3 阶段颜色

| 阶段 | 运行中颜色 |
|------|-----------|
| 感知层 | `var(--accent-amber)` |
| 视觉识别 | `var(--accent-green)` |
| RAG检索 | `var(--accent-blue)` |
| 决策生成 | `var(--accent-purple)` |

### 5.4 各阶段展开内容

**感知层**
- 标题：`感知层 · 帧提取`
- 运行中文字：`📷 正在从 DJI_0025_cut1.Mp4 提取帧... 第 N/M 帧`
- 完成摘要：显示帧数（如 `已完成 3 帧采样`）

**视觉识别**
- 标题：`视觉识别 · LLM 描述`
- 运行中文字：`🔍 正在分析航拍场景...`
- 完成内容：**LLM 输出经过总结后的 40-60 字摘要**（由后端总结，不截断）
- 若超时有报错：`⚠ LLM 推理超时，请检查 Ollama 服务`

**RAG检索**
- 标题：`RAG检索 · 规范匹配`
- 运行中文字：`📚 正在检索相关处置规范...`
- 完成内容：检索到的 1-2 条规范片段，每条约 80 字
- 若无结果：显示 fallback SOP

**决策生成**
- 标题：`决策生成 · AI 判定`
- 运行中文字：`⚡ 正在生成预警决策...`
- 完成内容：结构化预览
  ```
  标题：高速公路弯道违规停车
  风险：medium · 置信度 78%
  建议：立即通知交警，记录车牌
  ```

## 6. 交互流程

```
页面加载 → 获取 /api/v1/demo/thumbnail → 显示视频缩略图和名称
    ↓
用户点击"启动演示"
    ↓
按钮禁用，变为"◈ 运行中..."
    ↓
4 张卡片进入待机态（序号圆圈灰色）
    ↓
【感知层】卡片展开
  → SSE stage=perception → 显示"提取帧..."文字
  → 完成 → 卡片收起，显示摘要，绿勾
    ↓
【视觉识别】卡片展开
  → SSE stage=vision → 显示"正在分析..."文字
  → LLM 输出 → 后端总结 → 卡片收起，显示摘要，绿勾
  → 超时30秒 → 显示错误信息，卡片保留（可继续或等待5秒自动下一阶段）
    ↓
【RAG检索】卡片展开
  → SSE stage=rag → 显示"检索规范..."文字
  → 检索结果 → 卡片收起，显示片段，绿勾
    ↓
【决策生成】卡片展开
  → SSE stage=decision → 显示"生成决策..."文字
  → 决策结果 → 卡片收起，显示JSON预览，绿勾
    ↓
AlertCard 从下方滑入，显示最终预警结果
    ↓
15秒后 AlertCard 淡出，4张卡片全部重置为待机态，控制栏恢复
```

## 7. 后端改动

### 7.1 GET /api/v1/demo/thumbnail

**行为**：
- 从 `data/streams/DJI_0025_cut1.Mp4` 第 10% 帧提取一帧
- 压缩为 640×360 JPEG，保存到 `data/streams/.thumbnail_cache.jpg`
- 后续请求直接返回缓存文件

**返回**：`image/jpeg`

**错误**：`500`（视频不存在或无法读取）

### 7.2 GET /api/v1/demo/videos

**返回**：
```json
{
  "demo": "DJI_0025_cut1.Mp4",
  "available": ["DJI_0025_cut1.Mp4", "highway_aerial.mp4", ...]
}
```

注：当前仅支持 `DJI_0025_cut1.Mp4` 作为演示视频，其他视频可后台配置切换。

### 7.3 后端视觉识别总结

在 `routes_demo.py` 中，`_vision_describe` 返回后，调用 LLM 总结为 40-60 字：

**总结 Prompt**：
```
请将以下航拍场景描述总结为40-60字的简明摘要，保留关键信息：
{原始描述}
直接输出摘要，不要添加解释。
```

## 8. SSE 事件扩展

现有 `stage` 事件新增字段：

```json
{
  "stage": "vision",
  "progress": 25,
  "scene": "航拍俯视高速公路...",
  "summary": "主路通行正常，应急车道有施工车辆停靠"
}
```

| 字段 | 说明 |
|------|------|
| `stage` | 阶段名 |
| `progress` | 进度百分比 |
| `scene` | 原始场景描述（vision阶段有） |
| `summary` | 总结后的摘要（vision完成时有） |
| `error` | 错误信息（超时时包含） |

## 10. 设计原则

- **YAGNI**：不预设视频切换 UI，后续通过改配置文件即可
- **渐进增强**：视觉模型不可用时，照常走像素分析+fallback流程
- **容错友好**：阶段超时不跳过，显示报错，让用户决定下一步
- **视觉一致**：复用 `AlertCard` 组件风格，不引入新设计体系
